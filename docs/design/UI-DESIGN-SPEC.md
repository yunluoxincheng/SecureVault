# SecureVault UI 设计规范

> 版本 1.0 — 2026-03-18

---

## 一、设计原则

| 原则 | 说明 |
|------|------|
| **安全感** | 用户一眼就知道数据是受保护的——锁图标、加密状态提示、安全模式视觉区分 |
| **简洁** | 密码管理器的核心操作（查找→复制/填充）应在 2 次点击内完成 |
| **跨平台一致** | Android/iOS/Desktop 共享 80% UI 代码，视觉和交互逻辑一致 |
| **平台适应** | 尊重平台习惯——Android 返回手势、iOS 滑动返回、Desktop 键盘快捷键 |
| **无障碍优先** | 所有交互元素满足 WCAG 2.1 AA 标准 |

## 二、颜色体系

### 2.1 语义色（Light / Dark）

基于 Material Design 3 Dynamic Color，定义 fallback 色值：

| 语义角色 | Light 色值 | Dark 色值 | 用途 |
|---------|-----------|----------|------|
| **Primary** | `#1B6B4F` | `#7DDCB5` | 主按钮、选中状态、导航高亮 |
| **OnPrimary** | `#FFFFFF` | `#003828` | Primary 上的文字/图标 |
| **PrimaryContainer** | `#A4F4D2` | `#005138` | FAB 背景、卡片高亮 |
| **Secondary** | `#4A635A` | `#B3CCC1` | 次要按钮、标签 |
| **Surface** | `#F8FAF7` | `#191C1B` | 页面背景 |
| **SurfaceVariant** | `#DBE5DE` | `#404944` | 卡片背景、输入框背景 |
| **Error** | `#BA1A1A` | `#FFB4AB` | 错误提示、危险操作 |
| **Outline** | `#707973` | `#8A938D` | 边框、分割线 |

**选色理由**：绿色调传达"安全、可信赖"，避免过于鲜艳的蓝/紫（与普通工具类应用区分）。

### 2.2 功能色

| 用途 | 色值 | 说明 |
|------|------|------|
| 密码强度：非常弱 | `#D32F2F` | 红色 |
| 密码强度：弱 | `#F57C00` | 橙色 |
| 密码强度：一般 | `#FBC02D` | 黄色 |
| 密码强度：强 | `#388E3C` | 绿色 |
| 密码强度：非常强 | `#1B6B4F` | 深绿（=Primary） |
| 安全模式标记 | `#7B1FA2` | 紫色，安全模式条目的视觉区分 |
| 收藏标记 | `#FFB300` | 琥珀色星标 |

### 2.3 Dynamic Color 策略

| 平台 | 策略 |
|------|------|
| Android 12+ | 使用系统 Dynamic Color |
| Android 10-11 | 使用上述 fallback 色值 |
| iOS | 使用上述 fallback 色值（未来可适配系统色调） |
| Desktop | 跟随系统亮/暗模式，使用 fallback 色值 |

## 三、字体规范

### 3.1 字体族

| 平台 | 字体 |
|------|------|
| Android | Roboto (系统默认) |
| iOS | SF Pro (系统默认) |
| Desktop | 系统默认无衬线字体 |

不引入自定义字体，减少包体积并保持平台原生感。

### 3.2 文字样式

| 样式名 | 大小 | 行高 | 粗细 | 用途 |
|-------|------|------|------|------|
| `headlineLarge` | 28sp | 36sp | 400 | 页面标题（"密码库"、"设置"） |
| `headlineSmall` | 22sp | 28sp | 400 | 区块标题 |
| `titleLarge` | 20sp | 28sp | 500 | 卡片标题（密码条目名称） |
| `titleMedium` | 16sp | 24sp | 500 | 列表项标题 |
| `bodyLarge` | 16sp | 24sp | 400 | 正文、输入框文字 |
| `bodyMedium` | 14sp | 20sp | 400 | 次要信息（用户名、URL） |
| `bodySmall` | 12sp | 16sp | 400 | 标签、时间戳 |
| `labelLarge` | 14sp | 20sp | 500 | 按钮文字 |

### 3.3 密码显示字体

密码使用等宽字体以便辨认字符：

```kotlin

    Font(/* platform monospace font */)
)
// Android: "monospace", iOS: "Menlo", Desktop: "JetBrains Mono" / "Consolas"
```

## 六、动效规范

### 6.1 原则

- **有意义**：动效服务于操作反馈，不纯粹装饰
- **快速**：密码管理器的用户追求效率，动效不阻塞操作
- **平台一致**：使用 Compose Animation API，各平台表现一致

### 6.2 标准动效

| 场景 | 类型 | 时长 | 缓动 |
|------|------|------|------|
| 页面进入 | Fade + SlideIn | 300ms | EaseOut |
| 页面退出 | Fade + SlideOut | 250ms | EaseIn |
| 卡片出现（列表） | Fade + ScaleIn(0.95) | 200ms | EaseOut |
| 对话框弹出 | Fade + ScaleIn(0.9) | 250ms | EaseOutBack |
| 密码显示/隐藏 | CrossFade | 150ms | Linear |
| 复制成功反馈 | 图标旋转 + 颜色变化 | 300ms | EaseOut |
| 强度条变化 | 宽度 + 颜色渐变 | 400ms | EaseInOut |
| 解锁成功 | 锁图标 → 开锁动画 | 500ms | EaseOutBounce |
| 锁定 | Fade to 解锁界面 | 300ms | EaseIn |

### 6.3 不使用动效的场景

- 自动填充响应（系统控制 UI）
- 剪贴板操作（后台操作）
- 加密/解密过程（用加载指示器代替）

## 八、无障碍设计

### 8.1 基本要求

| 要求 | 实现 |
|------|------|
| 触摸目标 ≥ 48dp | 所有可点击元素 |
| 文字对比度 ≥ 4.5:1 | 正文与背景 |
| 大文字对比度 ≥ 3:1 | 标题与背景 |
| contentDescription | 所有图标和非文字元素 |
| 焦点顺序 | 从上到下，从左到右 |
| 键盘导航 | Desktop 全键盘可操作 |
| 屏幕阅读器 | TalkBack (Android) / VoiceOver (iOS) |

### 8.2 敏感信息处理

```kotlin
// 密码字段对屏幕阅读器隐藏具体内容
Modifier.semantics {
    contentDescription = "密码已隐藏，点击复制按钮使用"
    // 不读出密码明文
}

// 复制按钮的语义
Modifier.semantics {
    contentDescription = "复制密码到剪贴板，30秒后自动清除"
}
```

### 8.3 高对比度模式

提供高对比度配色方案，在系统辅助功能开启时自动切换：
- 背景全白/全黑
- 文字纯黑/纯白
- 边框加粗
- 强度指示器使用图案+颜色双重区分
